HTTP Status 500-Internal Server Error

用IntelliJ Idea开发项目时，发现了个奇怪的问题，还很恶心，500-Internal Server Error，就一个JSP页面打不开了，不知道为什么，在百度上查，各种查，但是查到的结果也是各有不同，500错误的原因好多，程序出错就会在页面出现个友好的界面错误提示：内部服务错误500，这就太坑了，我也没干啥啊，原先能点开的页面，现在点不开了，但是还提示The absolute url：[http://java.sun.com/jsp/jstl/core]cannot be resolved in either web.xml or the jar files deployed with this application.
各种错误提示如下面的图：



网上也好多解决方案，什么在WEB-INF文件夹下的lib文件夹下加上standard-1.1.2.jar和jstl-1.2.jar包，但在我这本来就有啊，还有说把<%@ taglib prefix=”c” uri=”http://java.sun.com/jstl/core “ %>改为<%@ taglib prefix=”c” uri=http://java.sun.com/jstl/jsp/core %>等吧，但是在我这就是没有，最后实在忍不住了，快要炸了的时候，发现我新建的JSP和以前的JSP界面开始不一样，以前的如下图：

新建的jsp如下图：

强制性删除新建的JSP的头，然后换成以前的JSP的头就可以了，问题解决，真坑。。。。。。要不是瞎换谁会这么干，或许还有其他的解决方案吧，我没再找了
